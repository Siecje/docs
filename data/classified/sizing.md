Размеры
=======

{{note функциональность не окончательна, грядут изменения }}


Конфигурирование
----------------

~~~js
var some = new webix.ui.view({
    width:100,    //ширина
    height:100,   //высота
    //gravity:5   //фактор автосайза
})
~~~


по умолчанию все компоненты пытаются автосайзится 






Паблик методы
-------------

some.resize(); запускает цикл простановки размеров, имеет смысл вызывать если были изменены какие то параметры конфигурации  

some.adjust(); пытается подогнать размеры вью к размерам внешнего html контейнера - вызывать особо не надо






Внутренняя логика
-----------------

Сначала для каждой компоненты вызывается $getSize который должен вернуть сколько компонента хочет занимать место

~~~js
$getSize:function(){
     return [gravityX, width, gravityY, height];
}
~~~

- gravityX - автосайз по оси х, 0 если не надо
- width - желаемая ширина, 0 если не надо ( но тогда должен быть выставлен автосайз ) 
- gravityY - автосайз по оси y, 0 если не надо
- height - желаемая высота, 0 если не надо ( но тогда должен быть выставлен автосайз ) 

чтобы учесть базовые настройки можно использовать синтаксис вида


~~~js
$getSize:function(){
     //set fixed widht, but allow other options
     this._settings.width = 200;
     
     return webix.ui.view.prototype.$getSize.call(this);
}
~~~


После того как желаемы размеры получены, будет вызван метод setSize с новыми размерами

~~~js
$setSize:function(x,y){
    if (webix.ui.view.prototype.$setSize.apply(this, arguments)){
        ...cusom_sizing...
    }
}
~~~

cusom_sizing блок будет вызван только если размеры поменялись после прошлой операции, внутри блока имеет смысл использовать _content_width, _content_height, вместо x и y ( так как x,y содержат бордеры ) 

Если компоненту устраивают дефолтные процессы простановки размеров - эти методы объявлять не надо






Внутреннее состояние
--------------------

#### this._content_width, this._content_height

размеры компоненты, без бордерной обводки и скролла - то есть реальное место под отрисовку компоненты

доступны извне как some.$width, some.$height

#### this._last_size

массив x,y из последних размеров

#### this._last_full_size

массив x,y из последних реальных размеров, в отличии от вышеуказанного включает бордеры, скролы, отступы и т.д.
